# Review Checklist

## Objective Checks

- correct document type was chosen
- central versus local placement is correct
- metadata fields are present
- status value is supported
- filename matches repo convention
- required headings exist
- internal links resolve

## Human Review Checks

- the decision is actually durable enough for an ADR
- the spec describes the intended behavior clearly enough
- a new file was created only when an existing one was not the right home
- consequences and non-goals are honest

## Enforcement Boundary

Automate only the objective checks.

Do not auto-block on prose style, completeness of reasoning, or architectural taste.
