# Extraction Matrix — SAASTools formalization -> Elegy package families

Scope: workflow formalization artifacts already changed/planned in SAASTools (workflow models/sharing/validation contract, WS3 governance scripts, agent-orchestration primitives).

Topology note: the current Elegy repo is package-family based. There is no catch-all `src/Elegy.Formalization/...` source tree. Extraction targets in this matrix therefore map to the real package families under `src/` (`Elegy.Formalization.Core`, `.Contracts`, `.Serialization`, `.Validation`, `.Governance`, `.Projections.Mermaid`, `.Skills`, `.Mcp`, `.Agents`, `.AgentFactory`, etc.).

Facade note: if downstream consumers later need a single convenience package, introduce it as a new explicit package family or metapackage after package-graduation review. Do not use synthetic paths as a placeholder for that future decision.
