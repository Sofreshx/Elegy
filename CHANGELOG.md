# Changelog

All notable changes to this project will be documented in this file.

The format is based on Keep a Changelog principles, adapted for the current bootstrap and consolidation stage.

## [Unreleased]

### Added

- **Central publish orchestrator.** `.github/workflows/publish-orchestrator.yml` is the only `publish-*` workflow in the repo. It walks `contracts/fixtures/elegy-plugin-package.*.json`, reads each `publishing` block, and dispatches a build per surface through `._reusable-publish.yml`. Adding a new publishable surface = add a fixture + nothing else; no per-feature workflow file, no central catalog to update. The orchestrator runs a `validate` job that executes the new conformance tests `plugin_package_publishing_blocks_have_orchestrator_required_fields` and `plugin_package_crate_paths_resolve_in_workspace` before any surface is built. The 11 per-feature `publish-elegy-*.yml` files are gone.
- **`elegy-cli contracts validate` subcommand.** New Rust-native subcommand that walks the contract bundle and validates every schema (parseable JSON), every fixture (parseable JSON + plugin-package conformance), and every per-feature `publishing` block (archive family, asset kind, crate path, skill bridge, installer consistency). Exits 0 on success, non-zero on any failure. Replaces the legacy `scripts/validate-canonical-outputs.ps1` and `scripts/export-contracts.ps1` flow; the central publish orchestrator and `package-boundaries.yml` both call it.
- **Two new conformance tests in `rust/core/elegy-contracts/tests/conformance.rs`.** `plugin_package_publishing_blocks_have_orchestrator_required_fields` enforces the central-orchestrator contract: every publishable fixture must declare `archiveFamily`, `assetKind`, and (for `assetKind=wrapper`) `skillBridge` + `installer`; `assetKind=cli` must declare `cratePath`. `plugin_package_crate_paths_resolve_in_workspace` walks the workspace's `members = [...]` list, resolves each member's `name` from its `Cargo.toml`, and asserts every fixture's `cratePath` resolves to a real workspace crate. Both tests run in CI on every push and on every release.
- **`publishing.cratePath` and `publishing.assetKind` in the `elegy-plugin-package/v1` schema.** Two new optional fields added to `contracts/schemas/elegy-plugin-package.schema.json` (and the `ElegyPluginPackagePublishingMetadata` Rust struct in lockstep). `cratePath` is the workspace crate name (`cargo build -p` target); `assetKind` is `cli` or `wrapper` and drives archive staging contents. Skill-only / external-CLI surfaces omit `cratePath`. All 12 publishable fixtures carry the new fields; 4 new fixtures (`elegy-cli`, `elegy-codegraph`, `elegy-memory-mcp-http`, `elegy-memory-mcp-stdio`) were added to cover the surfaces the previous per-feature workflows produced.
- governed contract bundle export and validation flows centered on the new `elegy-cli contracts export` and `elegy-cli contracts validate` subcommands (replaces the legacy PowerShell scripts)
- dedicated multi-OS Rust CI for the surviving workspace under `rust/`
- repository security workflow covering `cargo-deny`, `cargo-audit`, `gitleaks`, and Rust CodeQL analysis
- contributor-facing Rust CLI self-authoring slice for `author mcp`, `analyze mcp`, and `generate skills`, backed by `rust/core/elegy-tooling`
- `elegy-plugin-package` authoring template: `contracts/fixtures/elegy-plugin-package.template.json` is a new skeleton fixture with `<FILL-IN>` placeholders, schema-version pinned to `elegy-plugin-package/v1`, and an `_template_instructions` extension listing the steps to turn the skeleton into a valid package. `contracts/AGENTS.md` points at the template under an "Authoring a new package" section.
- `_reusable-publish.yml` plus a `release-finalize.yml` workflow for per-feature release asset assembly, checksum computation, and manifest emission.
- `release-finalize.yml` for the release: published hook: computes SHA-256 checksums and writes `elegy-release-manifest-<version>.json` + `elegy-release-checksums-<version>.json` into the release.
- `docs/governance/visual-llm-enforcement-policy.json` is the new home for the ws3 enforcement policy; `policies/formalization/` is gone.

### Changed

- **Repo URL corrected to the canonical `Sofreshx/Elegy`.** The workspace `repository` and `homepage` fields in `rust/Cargo.toml` previously pointed at `sfrsh/Elegy` while the workspace `documentation` field, the README, and the install instructions pointed at `Sofreshx/Elegy`. All four now point at the same canonical URL.
- **Five missing `publish = false` directives added.** `elegy-contracts`, `elegy-codegraph`, `elegy-memory`, `elegy-planning`, and `elegy-skills` previously inherited no `publish` setting and could be accidentally published to crates.io; they now match the rest of the workspace.
- **`.elegy/docs.yaml` schemaVersion aligned with the CLI's expected `elegy-docs/v1`.** The yaml file had drifted to the (unsupported by the current CLI) `elegy-documentation/v2`; the docs check rejected it on first run after the consolidation. The yaml now uses the v1 schema the CLI accepts.
- **Repository authority language now centers on `contracts/` (including `contracts/schemas/`) plus exported artifacts**, rather than removed source-package roots. Operational policy now lives at `docs/governance/`.
- **Architecture and policy docs now describe `rust/` as the active executable surface** and avoid claiming a built-in MCP-native or skill-driven self-authoring surface that the repo does not yet ship.
- **Validation posture now points to surviving scripts and workflows** instead of removed legacy build and test flows.
- **`contracts/AGENTS.md` updated to reflect the new auto-discovery publish model.** Per the user's choice at planning time, the central `publish-orchestrator.yml` walks fixtures and dispatches builds; the previous "each new feature needs its own `publish-<feature>.yml`" guidance is replaced by a one-paragraph "add a fixture, nothing else" instruction.
- **Spec roots consolidated**: the legacy `specs/` root has been folded into `docs/specs/`. The single `specs/neutral-package-consolidation/` record now lives at `docs/specs/neutral-package-consolidation/`. The global `AGENTS.md` redirect and the durable-spec convention are unchanged in spirit; only the physical path was unified.
- **`policies/` directory removed.** The single operational policy asset has been relocated to `docs/governance/visual-llm-enforcement-policy.json`; the workflow, the resolve script, and the README architecture and security posture have been updated to reference the new path.
- **`src/Elegy-*/install.ps1` wrapper installer lanes retired.** The seven parallel wrapper installers under `src/Elegy-*/` were thin passthroughs to `scripts/install-distribution.{ps1,sh}`. The plugin-package `elegy-plugin-package/v1` contract and the canonical installer with `-CliSurfaces` are now the install contract.
- **Rust workspace theme-grouped.** The flat `rust/crates/` workspace has been reorganized into three top-level trees that match the substrate doctrine: `rust/core/` for the shared kernel (9 crates: `elegy-contracts`, `elegy-policy`, `elegy-tooling`, `elegy-descriptor`, `elegy-adapter-fs`, `elegy-adapter-http`, `elegy-core`, `elegy-runtime`, `elegy-agent-events`), `rust/features/` for one tree per agent capability (18 crates), and `rust/bin/` for thin host and umbrella CLI entrypoints (`elegy-cli`, `elegy-host-mcp`). Crate names are unchanged, so `use elegy_xxx::...` paths in code are unaffected.
- **Feature file co-location.** Architecture mirrors and feature notes that previously lived at `docs/architecture/{elegy-configuration-v1,elegy-memory-v1,elegy-planning-v1,mermaid-tooling,observe-cli}.md`, `docs/research/elegy-memory-retention-removal.md`, and `docs/migration/reusable-memory-boundary.md` now sit next to their owning crate under `rust/features/<feature>/docs/`. Feature-specific specs (the 6 elegy-planning specs, the 2 codegraph specs, and the `agent-safe-planning-core/` spec+plan pair) moved into `rust/features/<feature>/docs/specs/`.
- **Distribution centralization refactor.** The previously monolithic `docs/distribution.md` (283 lines) and `PACKAGE_README.md` (58 lines) listed every binary's archive family inline. Each feature with a `[[bin]]` now owns its own `DISTRIBUTION.md` next to its crate. The top-level `docs/distribution.md` is a thin index. The 7×3 hardcoded `surface × target` matrix in `.github/workflows/distribution-artifacts.yml` was auto-derived from per-feature fixtures; the `publish-orchestrator.yml` introduced in this release supersedes it.
- **Per-plugin self-owned publishing.** Every plugin and feature now owns its own distribution end-to-end. The `elegy-plugin-package.<feature>.json` fixtures are the single source of truth for that feature's publish metadata. The central catalogs that previously hardcoded surface lists are gone: `scripts/install-distribution.sh` derives the surface list, binary names, asset prefixes, installer filenames, and skill bridge paths from the extracted contracts bundle's `contracts/fixtures/elegy-plugin-package.*.json` via `_load_surface_metadata_from_fixtures()`. A conformance test enforces the contract that every feature with a binary also declares its publish metadata.
- **Pre-existing dead diagnostic arm removed.** `ToolingError::InvalidGeneratorContract { path, issues }` was referenced in `rust/bin/elegy-cli/src/main.rs` but the variant did not exist; the match arm and its `CLI-GENERATOR-001` diagnostic are gone.
- **Pre-existing rustfmt drift absorbed.** `cargo fmt --all` was run once across the workspace; the CI `fmt` and `clippy` jobs keep the regression guard.
- **Pre-existing clippy hygiene pass.** `cargo clippy --workspace --all-targets -- -D warnings` was returning ~154 errors on `main`: 149 `clippy::unwrap_used` violations, plus 5 dead-code / let-unit-value / trait-method collisions. All are real fixes, not `#[allow(...)]` suppressions. `cargo clippy --workspace --all-targets -- -D warnings` now exits 0.
- **Centralized consolidation spec and ADR retired.** The `docs/specs/neutral-package-consolidation/{spec,plan}.md` migration spec and the `docs/adr/2026-06-16-elegy-plugin-package-v1-unification.md` decision record that superseded it have been removed. The unification is now a fact of the repo, not a record to argue.
- **Pre-existing CI break: `validate-registry-alignment` references removed.** `.github/workflows/rust-ci.yml`, `.githooks/pre-commit`, `.githooks/pre-push`, and `scripts/setup-hooks.ps1` referenced `scripts/validate-registry-alignment.ps1` after the script was deleted; the references are gone.
- **Stale archive deep-validation fixed.** `scripts/validate-canonical-outputs.ps1` `-RequireArchive` deep-validation block was asserting on the deleted `elegy-plugin-package-v2.*` and `skill-definition-v2.*` artifacts; the four entries are now current.
- **Stale plugin-package fixture fixed.** `contracts/fixtures/elegy-plugin-package.elegy-doc-practices.json` had a relative `documentationUri` that the URI validator rejected; the value is now the absolute URL matching the other plugin-package fixtures.
- **`install-distribution.sh` carries the canonical install contract** (1400+ lines, manifest-and-checksums-verified, surface metadata derived from per-feature fixtures). `install-distribution.ps1` is a 32-line shim that forwards all arguments to bash — no install logic of its own. The 11 wrapper install.ps1 files under `src/Elegy-*/` and the 7 hardcoded `surface × target` matrices in CI are all retired.

### Fixed

- **ADR broken internal links after spec migration.** `docs/adr/2026-06-15-adopt-elegy-planning-graph-core.md` linked to `docs/specs/elegy-planning*.md` paths that the consolidation moved to `rust/features/elegy-planning/docs/specs/`. The links are now re-pointed at the new feature-local paths. `elegy docs check` returns 0 errors and 0 warnings.
- **JSON-escape bug in `elegy-documentation-v2.schema.json`.** The `repoRelativePath.pattern` had `\.\.` in a JSON string (invalid: `\.` is not a legal JSON escape), which `elegy contracts validate` correctly rejected on first run. The escape is now `\\.\\.`, which decodes to the intended `\.` regex pattern.
- **Pre-existing test compile failure fixed.** A unit test in `rust/core/elegy-contracts/src/lib.rs` (`validate_elegy_plugin_package_rejects_unsafe_plugin_output_names`) was constructing an `ElegyPluginPackage` literal without the `extensions` field that was added to the struct after the test was written. The literal now includes `extensions: None`.

### Removed

- **Dead Holon-era and pre-consolidation artefacts swept.** The `holon-purge-{concepts,consumers,matrix}.md` migration docs (Holon migration residue), `docs/specs/host-neutral-plugin-install.md` (superseded by the `elegy-plugin-package/v1` contract and the per-plugin publishing path), and `docs/architecture/validation-plan-cleanup-2026-06-17.md` (a one-time validation plan for a now-complete cleanup pass) are gone. Their historical context is captured in this changelog; nothing in `docs/`, `rust/`, or `scripts/` still references them.
- **Orphaned schemas and fixtures with no real consumer.** `dynamic-skill-activation`, `elegy-catalog-entry`, `tool-projection`, `agent-create-request`, and `agent-definition` schemas (plus their minimal fixtures) had no live consumer in the Rust workspace or scripts and were removed. Schemas with real consumers (`capability-definition`, `summary-only-session-context-envelope`, `agent-capability-profile`, `agent-event-envelope`, `agent-request-envelope`, `agent-response-envelope`, `canonical-workflow*`, `elegy-plugin.lock`) stay.
- **Dead script tooling.** `scripts/setup-hooks.ps1` (now-redundant wrapper around a manual `git config`), `scripts/commit-check-{discover,run,setup}.mjs` (relied on a non-existent `package.json` and `.copilot/commit-checks.json`), `scripts/ws3/*` (only called by the ws3 governance workflow's own pre-resolved inputs, not by any other surface), `scripts/validate-canonical-outputs.ps1` (390 lines, replaced by `elegy-cli contracts validate`), `scripts/export-contracts.ps1` (PowerShell shim, replaced by `elegy-cli contracts export`), `scripts/package-installer.ps1` (PowerShell packager, now inlined into `distribution-artifacts.yml`), `scripts/discover-cli-surfaces.ps1` (PowerShell surface walker, now inlined into `publish-orchestrator.yml`), and `rust/.git-hooks/pre-push` (already redirected to `.githooks/pre-push` via comment header) are gone. Git hook setup is now a one-line `git config` documented in `CONTRIBUTING.md`. The 11 per-feature `publish-elegy-*.yml` files are gone; the central orchestrator dispatches all surfaces.
- **Empty and gitignored workspace residue.** The empty `tests/` directory at the repo root, the `artifacts/` directory of generated outputs (already covered by `artifacts/` in `.gitignore` and rebuilt from contracts on every release), and the gitignore entries for `FLIGHT_RECORDER.md`, `SESSION_TRACKING.md`, and the `/.copilot/` carve-out are gone. `FLIGHT_RECORDER.md` and `FLIGHT_RECORDER_PROTOCOL.md` (which were tracked despite being in `.gitignore`) are untracked and removed.
- **Stale agent-host instruction files.** `.github/instructions/elegy-docs.instructions.md` and `.github/instructions/elegy-memory.instructions.md` were redundant with the root `AGENTS.md` and `CLAUDE.md` and have been removed. Copilot-specific guidance is now consolidated in `.github/copilot-instructions.md`.
- **`.copilot/commit-checks.json` and its carve-out.** The Copilot commit-checks lanes pointed at the now-deleted `commit-check-*.mjs` scripts; `.githooks/pre-push` is the new single source of truth for "checks that must pass before push."
- **`.github/workflows/publish-distribution.yml`** (the old umbrella publish workflow) is gone; `publish-orchestrator.yml` + the per-feature fixture `publishing` blocks replace it.
